# Photo-Aiid-system 项目工作规范

## 自主工作模式（本项目专属，覆盖全局规范）

用户已授予本项目**完全的文件夹读写权限与自我判断权限**。

1. **无需逐次写操作前置确认**：在本项目内创建、修改、删除文件前，不必先输出"我计划修改以下文件"并等待确认，直接独立完成。
2. **独立工作与自检**：自行规划、实现、检测、修复。遇到可用合理默认值解决的问题，自行决策，不要打断用户提问。
3. **用户职责**：用户只负责最终成品的测试与验收。
4. **完成后汇报**：每轮工作结束后，列出实际修改的文件清单，便于用户审查。

## Git 操作（保留全局限制）

仍然**禁止**执行任何 git 命令（add / commit / push / merge）。
所有提交由用户通过 GitHub Desktop 手动审查后完成。

## 提交信息格式（供用户参考）

feat / fix / chore / docs / refactor + 简述
