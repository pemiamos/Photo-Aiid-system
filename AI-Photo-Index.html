v0.4 — 这一版能做什么

对应规格书 (Photo-Aiid-system) V1.0 完整闭环：挂载文件夹 → AI 自动打标 → EXIF 提取 → 动态索引表 → 模板化「物理」批量重命名 → CSV 导出。整理成果沉淀在文件名本身，不依赖软件存在。

相比 v0.3 Demo 的升级

  • 挂载文件夹：通过 File System Access API 非侵入式挂载本地文件夹（含子目录递归扫描），不复制、不上传。
  • 真·物理重命名：挂载模式下可直接改写磁盘上的文件名（Chrome / Edge）。执行后自动下载重命名日志 CSV（旧名 → 新名），可追溯。拖入的散图仍走 ZIP 副本。
  • 分析缓存：AI 结果按「路径+大小+修改时间」存入 IndexedDB，刷新页面、重新挂载同一文件夹时秒级恢复，不重复花费推理。
  • 恢复挂载：记住上次挂载的文件夹，下次打开一键恢复（需重新授权）。
  • 修复：序号在过滤视图下保持稳定；新文件名重名自动去重；语义搜索分批发送（大库不爆 token）；文件名/AI 输出统一转义。

引擎与环境要求

  • 物理重命名 / 挂载需要 Chrome 或 Edge（Safari、Firefox 不支持 File System Access API，自动回退到拖入 + ZIP 模式）。
  • Claude 引擎：本地用 Chrome 打开时需填写 API Key（console.anthropic.com 获取，仅存于本机浏览器 localStorage）；在 claude.ai Artifact 内可留空走免 Key 桥接，但 Artifact 内无法使用文件夹挂载与物理重命名。
  • Ollama 引擎：本地推理，符合「零云端」原则。终端执行 launchctl setenv OLLAMA_ORIGINS "*" 后重启 Ollama；模型需支持视觉（gemma3 / llava / qwen-vl 系列）。

与正式版的剩余差距

  • 语义搜索仍是「标签索引 + LLM 匹配」，正式版应迁移到本地向量库 embedding 检索。
  • 影子目录（虚拟分类的物理导出）、人脸聚类、OCR 入索引在后续版本。
  • 识别引擎正式版按规格迁移至 ONNX Runtime + CLIP，彻底脱离外部依赖。
