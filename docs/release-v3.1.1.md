# Photo-Aiid-system v3.1.1

AI 照片管理 + 网页征稿协作系统 —— v3.1 的稳定性与大库体验修复版（macOS / Apple Silicon · Windows x64）

## ✨ 这个版本修了什么

v3.1.1 是在 v3.1.0 基础上的**体验与稳定性修复版**，不改变功能范围，重点解决大库渲染卡顿、批量分析进度条提前消失，以及字段拖移的若干问题。

### 🖼️ 画廊解除 200 张上限，改为渐进式渲染
- **画廊不再只显示前 200 张**：上万张的大库现在会完整展示，改为分批渐进式渲染，滚动加载更顺滑，首屏不再卡死
- 缩略图懒加载继续生效，大库滚动时内存占用更平稳

### ⏳ 分析批次异常隔离，进度条不再提前消失
- **批量分析时单张出错不再拖垮整批**：worker 之间异常互相隔离，某一张失败会跳过并继续，不再让整个批次中断
- **进度条修复**：此前进度条会在任务真正完成前提前消失/归零，现已贴合实际处理进度，跑完才结束
- 重命名流程同步加固，异常处理更稳

### 🖱️ 字段拖移修复
- 侧栏命名字段、投稿管理书籍栏等处的**拖拽排序**修复，拖移更跟手、不再错位
- 索引表 / 管理面板的相关布局与交互一并打磨

> 本地照片整理（AI 识别、批量重命名、语义搜索、离线地理编码、拖拽到桌面等）与 v3.0/v3.1 的网页征稿、云端投稿管理、OSS/R2 双云存储**全部保留**。

## 📦 下载与安装

提供 **macOS（Apple Silicon）** 与 **Windows（x64）** 两个平台，按需下载。

### macOS（Apple Silicon M1/M2/M3/M4）
下载 `Photo-Aiid-system_3.1.1_aarch64.dmg`：

1. 双击 `.dmg`，将 `Photo-Aiid-system.app` 拖入「应用程序」文件夹
2. **首次打开会被 macOS 拦截**（提示“无法验证开发者”或“已损坏，无法打开”）——这是因为本应用未经苹果签名，**不是病毒**
3. 解决方法（任选一种）：
   - **右键点击 App 图标 → 打开 → 在弹窗中再次点击「打开」**（只需一次，之后可正常双击）
   - 或在终端执行：
     ```bash
     xattr -dr com.apple.quarantine "/Applications/Photo-Aiid-system.app"
     ```

### Windows（x64）
下载 `Photo-Aiid-system_3.1.1_x64-setup.exe`（NSIS 安装包，推荐）双击运行安装；另提供 `Photo-Aiid-system_3.1.1_x64_zh-CN.msi` 供企业部署。首次运行若被 SmartScreen 拦截，点「更多信息 → 仍要运行」即可（未签名所致，非病毒）。
