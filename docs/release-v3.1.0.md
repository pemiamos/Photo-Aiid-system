# Photo-Aiid-system v3.1.0

AI 照片管理 + 网页征稿协作系统 —— 从「单文件夹临时整理」升级为「多文件夹持久库 + 全局检索」（macOS / Apple Silicon · Windows x64）

## ✨ 这个版本带来了什么

v3.0 打通了本地整理与云端征稿；**v3.1 把本地整理侧从「一次只看一个文件夹」升级成「持久化的多文件夹照片库」，并新增跨全库的全局检索**。

### 🗂️ 多文件夹持久索引库
- **换文件夹不再清空历史**：每浏览/扫描一个文件夹就并入库，历史文件夹的元数据与 AI 标注**长期保留**；只要路径不变、已分析过的随时可被检索到
- **缩略图改为按需生成**：扫描阶段不再全量生成缩略图，首次被查看时才实时生成并缓存 —— 上万张规模也能轻量持有，磁盘占用极小
- **已索引文件夹注册表**：后端提供文件夹列表/移除/清空/清理失效路径的接口

### 🕘 索引表「历史索引」模式 ⭐ 重点推荐
> **最适合普通用户管理照片的功能。** 你不必记得照片存到了哪个文件夹——只要曾经分析过，就能在这里一次性搜出来。

- 索引表新增 **🕘 历史索引** 一键切换：**跨所有曾经索引过的文件夹**按关键词检索（摄影师 / 地点 / 标签 / 类别 / 文件名），等于给你的整个照片库建了一座可搜索的「历史档案馆」
- 典型场景：「去年在南京拍的那批」「张三投的稿」「所有带『日落』标签的」——输入关键词即可跨全库秒搜，不用再一个个文件夹翻找
- 搜索框下方两行快捷标签：**最近搜索关键词** + **全库最高频关键词**，点一下就搜
- 结果以**画廊同款瀑布卡片**展示，缩略图懒加载 + 分页「加载更多」，上万张大库也扛得住
- 操作按钮分两头排布更清晰：左侧「执行物理重命名 / 导出索引 CSV」，右侧「历史索引」

### 🖱️ 拖文件夹进画廊
- 直接把文件夹拖入画廊区即自动扫描并展示（桌面 App 可获取真实路径）

### 🛠️ 体验修复
- **摄影师识别**：原文件名中的署名（如 `张扬的小强-南京.jpg`）现优先于 AI 猜测，并支持更长昵称；相机自动文件名前缀（IMG/DSC/DJI 等）不再被误判为摄影师
- **重命名预览即时刷新**：在侧栏调整命名字段后，画廊文件名预览立即按新模板更新，并新增手动「刷新」按钮
- **库统计修复**：「已分析 / 标签数」此前因字段名错误恒为 0，现已正确统计
- **全局分析**可对已分析照片重复重新分析（带确认）
- **投稿管理 · 书籍栏**：超长列表竖向滚动、项目块拖拽排序、底部加「星尘远征队」署名
- **征稿地址**：粘贴整段投稿页/二维码链接（含 `?code=`）不再导致投稿码误判无效，并自动回填投稿码

> 本地照片整理（AI 识别、批量重命名、语义搜索、离线地理编码、拖拽到桌面等）与 v3.0 的网页征稿、云端投稿管理、OSS/R2 双云存储**全部保留**。

## 📦 下载与安装

提供 **macOS（Apple Silicon）** 与 **Windows（x64）** 两个平台，按需下载。

### macOS（Apple Silicon M1/M2/M3/M4）
下载 `Photo-Aiid-system_3.1.0_aarch64.dmg`：

1. 双击 `.dmg`，将 `Photo-Aiid-system.app` 拖入「应用程序」文件夹
2. **首次打开会被 macOS 拦截**（提示"无法验证开发者"或"已损坏，无法打开"）——这是因为本应用未经苹果签名，**不是病毒**
3. 解决方法（任选一种）：
   - **右键点击 App 图标 → 打开 → 在弹窗中再次点击「打开」**（只需一次，之后可正常双击）
   - 或在终端执行：
     ```bash
     xattr -dr com.apple.quarantine "/Applications/Photo-Aiid-system.app"
     ```

### Windows（x64）
下载 `Photo-Aiid-system_3.1.0_x64-setup.exe`（NSIS 安装包，推荐）双击运行安装；另提供 `Photo-Aiid-system_3.1.0_x64_zh-CN.msi` 供企业部署。首次运行若被 SmartScreen 拦截，点「更多信息 → 仍要运行」即可（未签名所致，非病毒）。
