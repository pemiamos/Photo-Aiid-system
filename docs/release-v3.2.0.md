# Photo-Aiid-system v3.2.0

AI 照片管理 + 网页征稿协作系统 —— 照片管理（相册 / 选片 / 整包下载）新增版（macOS / Apple Silicon · Windows x64）

## ✨ 这个版本新增了什么

v3.2.0 在征稿看板之外补上**照片的归集与导出**闭环，并加入**按书停止征稿**与**每本书独立投稿码**，让一本画册从收稿到选片落地全程顺畅。

### 🗂️ 新增「照片管理」：选片 → 相册 → 整包下载 ⭐ 重点
征稿看板只能看稿，这一版补上选片归档：

- **征稿看板可多选存片**：展开任一摄影师，点「选择照片」勾选若干张 →「存入文件夹」，选现有相册或当场新建。
- **新增「照片管理」标签页**（位于「征稿看板」前）：左侧是本书的相册列表（可新建 / 重命名 / 删除），右侧是相册内的照片墙。
- **整包下载 ZIP**：一键把相册里的全部原图实时打包成 zip 下到本地（用原始文件名，自动处理重名），直接进入排版 / 精修流程。
- 相册只是**对原照片的引用**：选片、移出、删相册都**不动原投稿照片**，可反复增减、零额外云存储占用；同一相册能装来自不同摄影师的照片。

### 🚀 征稿看板缩略图提速 ~10 倍
- 看板墙此前直接加载**原图**（单张 1～2 MB），一行九张并发拉取既慢又偶尔加载不全。现改为由 OSS 现成生成**缩略图**（单张约 10 KB，**约 11 倍省流**），秒开且稳定；点击放大仍是原图。

### 🛑 按书「停止征稿」
- 每本书的「投稿码管理」顶部新增**征稿开关**：项目做完点「停止征稿」，摄影师即无法再用投稿码校验或上传照片（投稿页提示「本书征稿已结束」）；可随时「恢复征稿」。
- 停止只挡新上传，**已收到的照片、看板、相册数据完全不受影响**。

### 🔢 每本书独立投稿码前缀
- 此前所有书共用一条 `A01、A02…` 全局序列，多本书的投稿码混在一起难以区分。现在**每本书分配独立前缀字母**（如 A、B、C…），投稿码形如 `B01、B02…`，各书分段、互不相撞；新建书自动领取下一个空闲字母。
- 老库自动迁移并去重（历史共用 A 的情况会自动分派到不同字母）。

> 本地照片整理（AI 识别、批量重命名、语义搜索、离线地理编码、拖拽到桌面等）与网页征稿、云端投稿管理、OSS/R2 双云存储**全部保留**。

## 📦 下载与安装

提供 **macOS（Apple Silicon）** 与 **Windows（x64）** 两个平台，按需下载。

### macOS（Apple Silicon M1/M2/M3/M4）
下载 `Photo-Aiid-system_3.2.0_aarch64.dmg`：

1. 双击 `.dmg`，将 `Photo-Aiid-system.app` 拖入「应用程序」文件夹
2. **首次打开会被 macOS 拦截**（提示“无法验证开发者”或“已损坏，无法打开”）——这是因为本应用未经苹果签名，**不是病毒**
3. 解决方法（任选一种）：
   - **右键点击 App 图标 → 打开 → 在弹窗中再次点击「打开」**（只需一次，之后可正常双击）
   - 或在终端执行：
     ```bash
     xattr -dr com.apple.quarantine "/Applications/Photo-Aiid-system.app"
     ```

### Windows（x64）
下载 `Photo-Aiid-system_3.2.0_x64-setup.exe`（NSIS 安装包，推荐）双击运行安装；另提供 `Photo-Aiid-system_3.2.0_x64_zh-CN.msi` 供企业部署。首次运行若被 SmartScreen 拦截，点「更多信息 → 仍要运行」即可（未签名所致，非病毒）。

> 升级提示：本版改动包含云端征稿后端（照片管理 / 停止征稿 / 投稿码前缀等接口）。如自建征稿服务器，请同步更新服务器上的 `backend/` 并重启 `photo-intake` 服务。
