# Photo-Aiid-system 使用教程

AI 智能照片管理与批量处理系统 —— macOS / Windows 桌面版使用手册。

---

## ⭐ 先看这个：历史索引 —— 找照片，不用再记它在哪个文件夹

> **这是最适合普通用户的功能。**

照片平时散在一堆文件夹里，时间一长根本记不清「那张照片放哪了」。只要照片**曾经被本软件分析过**，它就会一直留在「历史档案」里——打开 **「索引表」标签 → 右上角「🕘 历史索引」**，输入关键词就能**跨所有文件夹**一次性搜出来。

- 直接搜「**南京**」「**日落**」「**外婆**」——按 地点 / 标签 / 摄影师 / 文件名 秒搜
- 不想打字？搜索框下方有**最近搜过的词**和**整库最常见的标签**，点一下就搜
- 结果像相册一样瀑布平铺，缩略图随用随取，**几万张照片也不卡**

> 💡 从 **v3.1** 起，换文件夹**不会再清空**历史——你浏览、分析过的每个文件夹都会**累积进库**，用得越久、能搜的越多。要让某批照片可被搜到，记得先「浏览」并「分析」过它所在的文件夹。

## 1. 安装与首次启动

在 [GitHub Releases](https://github.com/pemiamos/Photo-Aiid-system/releases) 下载对应平台安装包：

**macOS（Apple 芯片）**
1. 打开 `Photo-Aiid-system_3.1.0_aarch64.dmg`。
2. 把 **Photo-Aiid-system** 图标拖到「应用程序 / Applications」文件夹。
3. 从「启动台」或「应用程序」里双击启动（首次打开见下一节绕过 Gatekeeper）。

**Windows（x64）**
1. 下载 `Photo-Aiid-system_3.1.0_x64-setup.exe`（NSIS 安装包，推荐）双击安装；企业部署另有 `_x64_zh-CN.msi`。
2. 首次运行若被 SmartScreen 拦截，点「更多信息 → 仍要运行」（未签名所致，非病毒）。

启动后约 1～2 秒会自动弹出主窗口（程序在后台启动内置服务，无需手动开启任何东西）。

> **系统要求**：macOS（Apple 芯片 / arm64）或 Windows 10/11（x64）。

## 2. ⚠️ 双击报错的解决办法（重要）

如果你是从别人那里拿到这个 App（而不是自己电脑构建的），**第一次双击大概率会报错**，例如：

- 「**无法打开"Photo-Aiid-system"，因为无法验证开发者。**」
- 「**"Photo-Aiid-system"已损坏，无法打开。**」
- 「**Apple 无法检查其是否包含恶意软件。**」

这**不是病毒，也不是程序坏了**，而是因为这个 App 没有经过 Apple 的付费签名/公证，macOS 的 Gatekeeper 默认拦截。任选下面一种方法解除：

### 方法 A：右键打开（最简单）
1. 在「应用程序」里找到 Photo-Aiid-system。
2. **按住 Control 键点击**（或右键）图标 → 选「**打开**」。
3. 弹窗里再点一次「**打开**」。
4. 之后就能正常双击了（只需做这一次）。

### 方法 B：系统设置里放行
1. 双击一次（会被拦截）。
2. 打开 **系统设置 → 隐私与安全性**。
3. 拉到底部，会看到「已阻止 Photo-Aiid-system…」，点旁边的「**仍要打开**」。

### 方法 C：终端去除隔离标记（适合「已损坏」提示）
如果上面都提示「已损坏」，在「终端」里执行（把路径换成实际位置）：

```bash
xattr -dr com.apple.quarantine "/Applications/Photo-Aiid-system.app"
```

执行后再双击即可。

> **彻底免除这些提示**需要开发者用 Apple 开发者账号（99 美元/年）对 App 做**签名 + 公证**。在那之前，发给别人时请连同本节说明一起发。

## 3. 接入 AI 引擎（API Key 配置）

软件本身**不含任何账号或密钥**，AI 识别能力需要你自己配置一个引擎。打开软件后，在**左侧栏「识别引擎」**下拉框选择引擎，并填入对应的 Key。**密钥只保存在你本机，不会上传到任何第三方。**

四种引擎，按需选一个即可：

### 🟢 智谱 GLM（推荐新手，有免费额度）
- 默认模型 `glm-4v-flash` —— **视觉识别免费**。
- 获取 Key：<https://open.bigmodel.cn/usercenter/apikeys>
- 在左侧栏选「智谱 / Zhipu」，把形如 `xxxxxxxx.xxxxxxxx` 的 Key 粘进「API Key」框。

### 🔵 Claude（识别精度最高，付费）
- 获取 Key：<https://console.anthropic.com>（控制台 → API Keys）。
- 选「Claude」，粘入形如 `sk-ant-…` 的 Key。
- 需联网，按用量计费。

### 🟡 Google Gemini（有免费额度）
- 默认模型 `gemini-2.5-flash`。
- 获取 Key：<https://aistudio.google.com/apikey>
- 选「Gemini」，粘入形如 `AIza…` 的 Key。

### ⚪ Ollama（完全本地、隐私优先、免 Key）
- 适合不想联网、不想花钱的场景，但需要你本机先装好 [Ollama](https://ollama.com) 并拉取一个**视觉模型**。
- 安装后在终端执行例如：`ollama pull gemma3:12b`（或其它视觉模型）。
- 选「Ollama」，Endpoint 默认 `http://localhost:11434`，在「视觉模型」里选你已安装的模型。
- 完全在本机运行，不需要 API Key，照片不出本地。

### 验证是否配好
填完 Key 后，点左侧栏底部的「**测试连接**」。显示成功即可开始分析。

## 4. 基本使用流程

1. **选择照片文件夹**：按界面提示选择一个包含照片的文件夹，软件会扫描并生成缩略图。
2. **配置引擎**：见上一节，选好引擎并填 Key，点「测试连接」。
3. **分析**：
   - 「**全局分析**」：分析当前文件夹里所有还没分析的照片。
   - 「**自选分析**」：先在画廊里点选若干张，再点此按钮，只分析选中的（含强制重分析）。
4. **查看与校正**：分析完成后会出现标签、地点等信息，可人工校正。
5. **批量重命名**：在左侧用字段块（如 `[摄影师]_[地点]_[标签]`）拼出命名规则，预览无误后执行重命名。
6. **导出原图**：在画廊中可把缩略图**直接拖拽到桌面或文件夹**，复制出（重命名后的）原图。
7. **导出索引 CSV**：在「索引表」一键导出，Excel / Numbers 可直接打开。
8. **历史索引** ⭐：跨所有分析过的文件夹一键找图——详见本文开头的重点介绍。这是日常找照片最常用的入口。

> 多文件夹：从 v3.1 起，依次「浏览 + 分析」多个文件夹，它们会**一起累积进库**（换文件夹不再清空），之后都能在「历史索引」里跨库搜到。画廊只显示当前文件夹，历史索引才跨全库。

## 5. 数据存放位置

打包版会把数据库和缩略图缓存放在用户可写目录（**不在 App 包内部**）：

```
macOS:   ~/Library/Application Support/Photo-Aiid-system/
Windows: %APPDATA%\Photo-Aiid-system\
├── photo_aiid.db        # 索引数据库（含设置 / API Key / 历史索引的全部记录）
└── thumbnails/          # 缩略图缓存
```

- **卸载/重置**：删掉上面这个文件夹即可清空所有索引与设置（历史索引也会一并清空）。
- **备份/迁移**：复制这个文件夹即可，连同历史索引一起搬到另一台电脑。
- 你的**原始照片不会被移动**到这里，软件只读取并生成缩略图；重命名是在原位置进行。

## 6. 常见问题

**Q：打开后窗口一直空白/转圈？**
A：内置服务启动需要 1～2 秒。如果长时间空白，多半是 8000 端口被别的程序占用。关掉占用 8000 的程序后重开。

**Q：提示"无法连接后端服务"？**
A：同上，通常是端口冲突或内置服务异常退出。完全退出 App（Cmd-Q）后重开。

**Q：分析报错 "API key not configured"？**
A：当前引擎没填 Key，或填错。回到左侧栏检查并「测试连接」。

**Q：CLIP 引擎显示不可用？**
A：CLIP 是可选的本地模型，打包版为控制体积**未内置**。请改用智谱/Gemini/Claude/Ollama。

**Q：换了引擎后没反应？**
A：切换引擎后记得填该引擎对应的 Key 并「测试连接」。不同引擎的 Key 各自独立保存。

**Q：关掉窗口程序就退出了？**
A：是的，这是设计行为——关窗即完全退出并停止后台服务，避免残留进程占用端口。
