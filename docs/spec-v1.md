# 技术规格书：Photo-Aiid-system（AI 智能照片管理与批量处理桌面应用）

## 1. 项目愿景
开发一款轻量级、隐私优先（全本地运行）的桌面照片管理工具。用户只需“链接”文件夹，应用即可通过 AI 自动完成照片内容识别、元数据提取、语义打标，并自动生成关键词索引目录，支持物理/虚拟的批量重命名与分类。

---

## 2. 核心功能模块 (Core Features)

### A. 智能扫描与挂载 (Non-destructive Sync)
*   **非侵入式链接：** 应用不拷贝原始文件，仅读取路径。
*   **增量监控：** 实时监控已链接文件夹，自动分析新加入的照片。
*   **EXIF 提取：** 自动提取拍摄日期、相机型号、镜头参数、GPS 坐标。

### B. AI 语义识别引擎 (AI Vision Engine)
*   **多维自动打标：** 基于 **CLIP (Contrastive Language-Image Pre-training)** 模型，实现跨模态识别（自然风光、人像、美食、文档、宠物等）。
*   **中英双语对齐：** 后台使用英文 Prompt 推理以确保高精度，前端显示用户友好的中文标签。
*   **语义特征化：** 将图片转换为向量（Vector Embedding），存入本地数据库，为语义搜索做准备。

### C. 自动化索引与重命名 (Batch Engine)
*   **动态索引表：** 实时生成包含“原文件名、AI标签、拍摄日期、建议新名”的索引目录。
*   **批量重命名逻辑：** 支持用户自定义模板，如：`[前缀]_[AI标签]_[拍摄日期]_[序号].jpg`。
*   **影子目录：** 支持在不移动原文件的前提下，创建基于标签的虚拟文件夹结构。

### D. 语义检索系统 (Search & Retrieval)
*   **多维过滤器：** 按日期、标签、器材、文件大小进行筛选。
*   **自然语言搜索：** 支持用户输入“海边日落时的合照”等描述性词汇寻找照片（语义搜索）。

---

## 3. 技术栈建议 (Recommended Tech Stack)

| 维度 | 推荐技术 | 说明 |
| :--- | :--- | :--- |
| **前端框架** | **Electron** + **React/Vue** | 跨平台兼容性好，能实现现代化的 UI 设计。 |
| **本地推理** | **ONNX Runtime** / **OpenVINO** | 确保 AI 模型在没有独立显卡的普通电脑上也能快速运行。 |
| **AI 模型** | **CLIP (ViT-B-32)** | 兼顾体积与精度，适合本地语义识别。 |
| **本地数据库** | **SQLite (w/ Vector Extension)** | 存储元数据及向量特征，支持百万级图片秒开。 |
| **图像处理** | **OpenCV** / **Sharp** | 负责缩略图生成及批量重命名/移动操作。 |

---

## 4. 经过 Demo 验证的关键逻辑

1.  **AI 识别优化：** 必须使用英文 Prompt（如 "a portrait of a person"）作为推理中介，以克服模型对中文理解的偏差。
2.  **缓存机制：** 必须对扫描结果进行持久化存储。第一次扫描下载模型较慢，后续需实现秒级启动。
3.  **UI 交互：** 采用“左侧设置面板 + 顶部 Tab 标签页 + 下方瀑布流画廊”的布局，用户体验最直观。

## 5. 后续改进与版本规划 (Roadmap)

### 第一阶段 (V1.0)：基础自动化
*   实现稳定的本地文件夹挂载与全自动识别分类。
*   实现索引表 CSV 导出。
*   实现一键物理批量重命名。

### 第二阶段 (V2.0)：深度语义与安全
*   **本地人脸聚类：** 识别特定人物并自动聚合。
*   **OCR 文字识别：** 识别照片中的路牌、文档内容，并加入索引。
*   **私密相册：** 对特定照片进行本地加密隐藏。

### 第三阶段 (V3.0)：智能编辑与增强
*   **AI 批量修图：** 根据照片类型自动调整光影（如美食照片自动调高饱和度）。
*   **查重与去重：** 自动识别内容高度相似的照片，建议用户清理无用素材。
*   **语义搜索增强：** 接入更强大的大模型（如本地运行的 LLaVA），支持更复杂的自然语言查询。

---

## 6. 性能与隐私规范
*   **零云端依赖：** 所有图片识别过程必须在用户本地 CPU/GPU 完成，严禁上传云端。
*   **低内存占用：** 针对数万张图片的库，需使用流式读取和分页显示（Lazy Loading）。
*   **模型精简：** 正式版需将模型文件（约 600MB）打包在安装包内或提供一键静默下载。
