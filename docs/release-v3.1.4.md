# Photo-Aiid-system v3.1.4

AI 照片管理 + 网页征稿协作系统 —— 类别细化为二级子类（macOS / Apple Silicon · Windows x64）

## ✨ 这个版本带来了什么

### 🏷️ 类别细化：主类 → 主类·子类 ⭐
此前 AI 类别只有一个大类（如 `自然风光`、`物种`、`建筑`）。现在升级为**两级**，用「·」连接，分类更精准：

- `自然风光·日落`、`人像·儿童`、`建筑·古建筑`、`街拍·夜市`、`美食·甜点`
- 物种保持主类「物种」不变，新增子类：`物种·猛禽`、`物种·鸣禽`、`物种·菊科` 等（**不影响原有的「物种」分组与书籍栏**）
- 文件名有具体地名/事件时，主类仍优先采用它，子类补更细的场景/主题
- 确实无法细分时只输出主类、省略「·子类」

配套：4 个引擎（Claude / 智谱 / Gemini / Ollama）的提示词已统一抽到共享模块维护，行为一致。

> **重要：类别是 AI 分析结果，升级后需要对照片重新分析才会出现二级子类。** 对已分析照片可用「全局分析」重跑（带确认）；新照片直接生效。
>
> 注意：类别会进入重命名模板的 `[类别]` 字段，启用后物理重命名的文件名会带上「·子类」（如 `杨建发_自然风光·日落_20210626.jpg`）。如不想要可在侧栏命名字段里移除「类别」。

### 📦 含 v3.1.1 ~ v3.1.3 的全部修复
- 原始文件名永久保留 + 历史索引检索覆盖原名/地点/摄影师
- 大图 / HEIC 照片正常出缩略图与 AI 检测
- 跳过并清理 macOS `._xxx` 伴随文件
- 画廊解除 200 张上限、渐进式渲染；分析批次异常隔离；进度条/字段拖移修复
- 顶部徽章自动显示真实版本号

## 📦 下载与安装

提供 **macOS（Apple Silicon）** 与 **Windows（x64）** 两个平台，按需下载。

### macOS（Apple Silicon M1/M2/M3/M4）
下载 `Photo-Aiid-system_3.1.4_aarch64.dmg`：

1. 双击 `.dmg`，将 `Photo-Aiid-system.app` 拖入「应用程序」文件夹
2. **首次打开会被 macOS 拦截**（提示“无法验证开发者”或“已损坏，无法打开”）——这是因为本应用未经苹果签名，**不是病毒**
3. 解决方法（任选一种）：
   - **右键点击 App 图标 → 打开 → 在弹窗中再次点击「打开」**（只需一次，之后可正常双击）
   - 或在终端执行：
     ```bash
     xattr -dr com.apple.quarantine "/Applications/Photo-Aiid-system.app"
     ```

### Windows（x64）
下载 `Photo-Aiid-system_3.1.4_x64-setup.exe`（NSIS 安装包，推荐）双击运行安装；另提供 `Photo-Aiid-system_3.1.4_x64_zh-CN.msi` 供企业部署。首次运行若被 SmartScreen 拦截，点「更多信息 → 仍要运行」即可（未签名所致，非病毒）。
