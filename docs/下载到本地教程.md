# 把征稿照片下载到本地

> 征稿期间编辑随时要把照片拉到本机选图、排版。本教程给命令行(rclone)与图形界面(Cyberduck)两种方式。
> **省钱关键：优先从 R2 下载(下行流量免费)，少从 OSS 下载(每 GB 约 0.5 元公网流量费)。**
> 前提是 R2 已通过 [sync-to-r2.sh](../scripts/sync-to-r2.sh) 持续同步(见 [征稿工作流程.md](征稿工作流程.md))。

---

## 方式一 · 命令行 rclone（推荐，最快）

已配好 `~/.config/rclone/rclone.conf`（remote 名 `oss`、`r2`）。`copy` 只下载本地缺的/有变化的文件，可反复跑做增量同步。

### 从 R2 下载（免费流量，推荐）

```bash
# 下载整本到本地目录
rclone copy r2:photo-archive/2026-sanxia ~/三峡画册 --transfers 8 --progress

# 只下载某位摄影师（投稿码-姓名）
rclone copy "r2:photo-archive/2026-sanxia/A01-张伟" ~/三峡画册/A01-张伟 --progress

# 只下载某个内容标注分类（如 获奖）
rclone copy "r2:photo-archive/2026-sanxia" ~/三峡画册 --include "*/获奖/**" --progress
```

### 从 OSS 直接下载（会产生公网流量费，按需用）

把上面命令里的 `r2:photo-archive` 换成 `oss:photo-intake` 即可。例：

```bash
rclone copy oss:photo-intake/2026-sanxia ~/三峡画册 --transfers 8 --progress
```

### 常用辅助

```bash
rclone ls   r2:photo-archive/2026-sanxia          # 列出所有文件+大小
rclone size r2:photo-archive/2026-sanxia          # 看这本书一共多大（先估下载量）
rclone tree r2:photo-archive/2026-sanxia          # 树状看目录结构
```

> `copy` 不删本地已有文件，安全；想让本地与云端完全一致（含删除）才用 `sync`，**别对反方向用 sync 以免误删云端**。

## 方式二 · 图形界面 Cyberduck（适合非技术同事）

[Cyberduck](https://cyberduck.io/)（免费，Mac/Win）支持 S3 协议，能直接连 R2 或 OSS，像网盘一样拖拽下载。

**连 R2（推荐）：**
1. 新建书签 → 类型选 **Amazon S3**。
2. Server 填：`79f77b78b3ef9c11881e4a6e380bdcf1.r2.cloudflarestorage.com`
3. Access Key ID / Secret 填 R2 的 S3 凭证（即 rclone.conf 里 `[r2]` 那对）。
4. 连上后进入 `photo-archive/2026-sanxia`，选中文件夹拖到本地即可。

**连 OSS：** Server 填 `oss-cn-hangzhou.aliyuncs.com`，凭证用 `[oss]` 那对，进 `photo-intake/2026-sanxia`。

> 也可用阿里云官方 **ossbrowser** 或 OSS 控制台「文件列表」逐个/打包下载，但都走 OSS 公网流量计费。

---

## 方式三 · 阿里云控制台（临时看几张）

OSS 控制台 → 桶 `photo-intake` → 文件列表 → 进入 `2026-sanxia/...` → 勾选 → 下载。
适合临时核对个别文件；批量下载请用 rclone/Cyberduck，控制台逐个点效率低。

---

## 费用提醒

| 来源 | 下行流量费 | 建议 |
|---|---|---|
| **R2** | **免费** | 日常批量下载都走这里 |
| OSS | ~0.5 元/GB | 仅在 R2 还没同步到、或要拿原始上传副本时用 |

下载前可先 `rclone size r2:photo-archive/2026-sanxia` 估算数据量。
