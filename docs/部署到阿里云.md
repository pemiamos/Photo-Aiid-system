# 把征稿后端部署到云服务器（拿固定投稿地址）

摄影师投稿需要一个**公网能打开的固定地址**。照片本来就是从浏览器**直传阿里云 OSS**，
不经过这台服务器；服务器只跑一个很轻的「投稿网页 + 验码/签名/看板」小服务。把它放到
一台云服务器上，就能得到 `https://你的域名/intake` 这样固定的投稿地址，不依赖你本机、
不用隧道。

> 桌面 App 的画廊、AI 分析等仍在你本机，不受影响。App 的「投稿管理」会远程连这台服务器。

---

## 一、买一台服务器（约 ¥30–60/月）

推荐 **阿里云轻量应用服务器**，最低配（2 核 2G）足够。

- **地域**：
  - **香港/新加坡等海外节点（推荐）** —— **免 ICP 备案**，可直接用域名 + HTTPS，部署最顺。
    照片仍直传杭州 OSS（摄影师上传速度不受影响，API 流量很小）。
  - 大陆节点（杭州，和 OSS 同区延迟最低）—— 用域名跑网站**需要 ICP 备案**（约 1–2 周）。
    没备案前可先用「公网 IP + HTTP」临时方案（见下）。
- **镜像**：Ubuntu 22.04 LTS。
- **安全组/防火墙**：放行 **22（SSH）、80（HTTP）、443（HTTPS）**。

买好后记下**公网 IP**，并设置好 root/普通用户的 SSH 登录。

## 二、准备域名（用 IP 临时测试可跳过）

本项目已有域名 **`sdexp.org`（托管在 Cloudflare）**，用一个子域 `intake.sdexp.org` 即可。

1. 进 Cloudflare → 选 `sdexp.org` → 左侧 **DNS → Records → Add record**：
   - 类型 `A`、名称 `intake`、IPv4 填服务器**公网 IP**。
   - **Proxy status 选「DNS only」（灰云 ☁️，不要橙色代理）** —— 关键！灰云让域名直接解析
     到服务器，下面的 `certbot` 才能直接签 HTTPS 证书。橙云（CF 代理）回源/证书更复杂，
     先别开；等跑通了想用 CF 加速再单独折腾。
2. 等解析生效：`ping intake.sdexp.org` 能看到服务器 IP 即可。

> 备案提醒：域名在 Cloudflare **不影响**备案规则，备案看**服务器位置**。要立刻用上
> `intake.sdexp.org` + HTTPS，请在第一步**买香港/海外节点**（免备案）；大陆节点需先备案。

## 三、上传后端代码

把仓库里的整个 `backend/` 目录传到服务器，例如 `/opt/photo-intake`：

```bash
# 在你本机仓库根目录执行（把 user@IP 换成你的）
rsync -av --exclude '__pycache__' --exclude '*.db' --exclude 'intake_data' \
  backend/ user@服务器IP:/opt/photo-intake/
# 顺带把 deploy/ 与 scripts/ 也传上去（部署脚本要用）
rsync -av deploy scripts user@服务器IP:/opt/photo-intake/../photo-intake-repo/ 2>/dev/null || true
```

简单起见，也可以直接在服务器上 `git clone` 整个仓库，则代码目录就是仓库里的 `backend`，
`deploy/`、`scripts/` 也都齐了。

## 四、填写 `.env`

在代码目录（`/opt/photo-intake`）下新建 `.env`，至少包含：

```ini
# 管理口令：桌面 App 远程管理用，务必设一个长随机串
INTAKE_ADMIN_TOKEN=换成你的长随机口令

# 阿里云 OSS 直传（和你本机 .env 同值即可）
OSS_REGION=oss-cn-hangzhou
OSS_BUCKET=photo-intake
OSS_ACCESS_KEY_ID=你的子账号AK
OSS_ACCESS_KEY_SECRET=你的子账号SK
OSS_STS_ROLE_ARN=acs:ram::1654081467881220:role/oss-intake-upload
```

> OSS 的 CORS 之前已配为允许来源 `*`，新域名无需再改。

## 五、一键部署

在代码目录的上层（能看到 `scripts/deploy-server.sh` 的地方）执行：

```bash
# 有域名（推荐，自动签 HTTPS）：
sudo bash scripts/deploy-server.sh intake.sdexp.org /opt/photo-intake

# 仅用公网 IP + HTTP 临时测试（不签证书、暂不上域名）：
sudo bash scripts/deploy-server.sh - /opt/photo-intake
```

脚本会自动：装依赖 → 建 venv 装精简依赖 → 起 systemd 常驻服务 →
配 nginx 反代 →（有域名时）certbot 申请 HTTPS。

完成后会打印投稿页地址与「App 征稿服务器地址」。

## 六、验证

```bash
curl -s http://127.0.0.1:8000/healthz        # {"ok":true}
```

浏览器打开 `https://intake.sdexp.org/intake` —— 应显示投稿页。

回到桌面 App →「投稿管理」→ 填 **征稿服务器地址 = `https://intake.sdexp.org`** +
**管理口令** → 连接成功后即可建书、发投稿码、看看板。投稿地址也会显示在书籍头部，
一键复制发给摄影师。

## 七、日常运维

```bash
sudo systemctl restart photo-intake     # 改了 .env 后重启
journalctl -u photo-intake -f           # 看实时日志
```

- **证书续期**：certbot 自动续期，无需手动。
- **归档到 R2**：仍在你本机跑 `bash scripts/archive-to-r2.sh <书目代号>`
  （需要本机 rclone 能读云端同一个 OSS 桶）。

## 常见问题

- **打不开 / 502**：`journalctl -u photo-intake -f` 看后端是否起来；`sudo nginx -t` 查配置。
- **证书申请失败**：确认域名已解析到本机、80 端口安全组放行；大陆节点需先备案。
- **App 连接失败**：地址要带 `https://`、结尾不要多斜杠；口令需与服务器 `.env` 的
  `INTAKE_ADMIN_TOKEN` 完全一致。
- **安全**：这台服务器只暴露 `/intake` 与 `/api/intake/*`，没有画廊/照片接口；管理接口
  靠口令头鉴权。请妥善保管 `INTAKE_ADMIN_TOKEN`。
