## 一、阿里云一次性配置（每个环境一次）

### 1. 建 OSS 桶 + CORS
1. OSS 控制台 → 创建 Bucket：名 `photo-intake`，地域 `oss-cn-hangzhou`，读写权限 **私有**。
2. 桶 → 数据安全 → 跨域设置 → 创建规则：
   - 来源：联调 `*`；**上线改成正式域名**（如 `https://你的域名`）
   - Methods：`GET POST PUT`
   - Headers：`*`
   - 暴露 Headers：`ETag`、`x-oss-request-id`
   - 缓存：`600`

### 2. 建 RAM 角色（被扮演）
1. RAM → 身份管理 → 角色 → 创建角色：可信实体 **阿里云账号 → 当前云账号**，名 `intake-oss-writer`。
2. 为角色授权：系统策略 `AliyunOSSFullAccess`（临时凭证会被后端进一步收窄到前缀）。
3. 复制角色 ARN。

### 3. 建 RAM 用户（后端发凭证用）
1. RAM → 身份管理 → 用户 → 创建用户：名 `intake-backend`，访问方式勾 **使用永久 AccessKey 访问**。
2. **当场保存 AccessKey ID / Secret（只显示一次）**。
3. 为用户添加权限：系统策略 `AliyunSTSAssumeRoleAccess`。

> 安全：AK/SK 仅放服务器环境变量，绝不进前端、不进安装包、不进 git。

## 二、启动后端

```bash
cd /Users/amos/Documents/GitHub/Photo-Aiid-system/backend

export OSS_REGION=oss-cn-hangzhou
export OSS_BUCKET=photo-intake
export OSS_ACCESS_KEY_ID=你的AK
export OSS_ACCESS_KEY_SECRET=你的SK
export OSS_STS_ROLE_ARN=acs:ram::1654081467881220:role/intake-oss-writer
export INTAKE_ADMIN_TOKEN=后台管理口令

.venv/bin/python -m uvicorn main:app --port 8000
```

环境变量速查：

| 变量 | 说明 | 缺失后果 |
|---|---|---|
| `OSS_REGION` | 带 `oss-` 前缀的地域 | 回退本地直存模式 |
| `OSS_BUCKET` | 桶名 | 同上 |
| `OSS_ACCESS_KEY_ID/SECRET` | 后端用户 AK/SK | 同上 |
| `OSS_STS_ROLE_ARN` | 被扮演角色 ARN | 同上 |
| `INTAKE_ADMIN_TOKEN` | 管理后台口令 | 不设则后台对任何人开放（仅限本机调试） |
| `INTAKE_BOOK` / `INTAKE_BOOK_TITLE` | 书目代号 / 书名（可选） | 默认 `2026-sanxia` / `三峡画册` |

## 三、每本书的运营流程

### 1. 建码、发链接（编辑）
- 打开 `/intake/admin/codes` → 输入口令登录。
- 「批量新增」：粘贴一列摄影师姓名，自动连号 A01、A02…
- 每行「复制」按钮 → 得到投稿链接 `https://你的域名/intake?code=A01` → 发给对应摄影师。

### 2. 摄影师投稿（零安装）
点链接（自动带入投稿码）→ 拖入照片 → 框选同类 → 批量标注内容（**标注完才能交**）→ 勾版权授权 → 提交。文件分片直传 OSS。回访可见历史批次、继续追加。

### 3. 跟进（编辑）
- `/intake/admin` 看板：谁交了 / 未交、各多少张、内容标注分布 → 据此催稿。

### 4. 取用与归档
- 编辑端把照片拉回本地 → 接现有 AI 分析 / 重命名流程。
- 截稿后归档到 Cloudflare R2（先配好 `scripts/rclone.conf.example`）：
  ```bash
  scripts/archive-to-r2.sh 2026-sanxia          # 导出元数据→copy→check，OSS 原样保留
  scripts/archive-to-r2.sh 2026-sanxia --purge  # 核对通过+二次确认后清空 OSS
  ```

## 四、联调自检清单

1. `GET /api/intake/oss-config` → `{"mode":"oss",...}`（不是 `local`）。
2. 打开 `/intake?code=A01`，上传 1～2 张。
3. F12 → Network：能看到 `sts`(200) → 直打 `{bucket}.{region}.aliyuncs.com` 的
   `?uploads=` / `?partNumber=` / `?uploadId=`（Initiator 为 `aliyun-oss-sdk`）→ `record`(200) → `complete`(200)。
   **这些上传请求不经过后端**即为正确。
4. OSS 控制台桶内出现 `2026-sanxia/A01-姓名/{标注}/文件名`。
5. `/intake/admin` 看板该摄影师张数增加。

常见报错：
- Network 里 OSS 请求 CORS 失败 → 桶跨域规则没配好 / 来源不含当前域名。
- `STS 获取临时凭证失败` → 角色 ARN、用户的 AssumeRole 权限或 AK/SK 有误。
- 上传 403 → 角色对桶无写权限，或 object_key 越权（不在本人前缀内）。

## 六、本地回退模式（无需 OSS）

不设 OSS 环境变量时，系统自动用「文件经后端落本地 `backend/intake_data/`」模式，
目录结构与 OSS 完全一致。适合无网/演示/快速验证；`backend/intake_data/` 已被 gitignore。
